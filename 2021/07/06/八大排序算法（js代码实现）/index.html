八大排序算法（js代码实现）

图片名词解释：

  • n: 数据规模
  • k: “桶”的个数
  • In-place: 原位操作，即只需用到O(1)的额外空间，不占用额外内存
  • Out-place: 占用额外内存

算法的分类

  • 插入排序
    • 直接插入排序
    • 希尔排序
  • 选择排序
    • 简单选择排序
    • 堆排序
  • 交换排序
    • 冒泡排序
    • 快速排序
  • 归并排序
  • 基数排序

算法的稳定性

  • 概念

    在待排序的记录序列中，存在多个具有相同的关键字的记录，若经过排序，这些记录的相对次序保持不变，即在原序列中。

    例如：r[i] = r[j]，且r[i]在r[j]之前，而在排序后的序列中，r[i]仍在r[j]之前，则称这种排序算法是稳定的；否则称为不稳定的。

  • 归类

    • 不稳定：选择排序、快速排序、希尔排序、堆排序
    • 稳 定：冒泡排序、插入排序、归并排序、基数排序

  • 好处

    从一个键上排序，然后再从另一个键上排序，第一个键排序的结果可以为第二个键排序所用。这样，有可能减轻系统的开销

1. 冒泡排序（Bubble Sort）

1.1 算法概述

这个算法的名字由来: 因为越大的元素会经由交换慢慢“浮”到数列的顶端（升序或降序排列），就如同碳酸饮料中二氧化碳的气泡最终会上浮到顶端一样，故名“冒泡排序”

  • 比较相邻的元素。如果第一个比第二个大，就交换它们两个；
  • 对每一对相邻元素作同样的工作，从开始第一对到结尾的最后一对，这样在最后的元素应该会是最大的数；
  • 针对所有的元素重复以上的步骤，除了最后一个 (len-1)；
  • 重复步骤1~3，直到排序完成。

1.2 算法实现

function bubbleSort(arr) {
    const len = arr.length;

    for (let i = 0; i < len; i++) {
        for (let j = 0; j < len - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
            }
        }
    }
    return arr;
}

1.3 算法分析

  • 最佳情况：T(n) = O(n)
  • 最差情况：T(n) = O(n^2)
  • 平均情况：T(n) = O(n^2)

1.4 算法优化

  • 设置一个flag

    用于标识一趟序列遍历(j循环) 中元素没有发生交换，则证明该序列已经有序，结束整个双重循环

    时间复杂度不变，但比较次数减少

2. 选择排序（Selection Sort）

2.1 算法概述

表现最稳定的排序算法之一，因为无论什么数据进去都是O(n2)的时间复杂度，所以用到它的时候，数据规模越小越好。理论上讲，选择排序可能也是平时排序一般人想到的最多的排序方法

  • 首先在未排序序列中找到最小（大）元素，存放到排序序列的起始位置
  • 然后，再从剩余未排序元素中继续寻找最小（大）元素，然后放到已排序序列的末尾
  • 以此类推，直到所有元素均排序完毕
  • ( 就是先找第一小的，再找第二小的，再找第三小的，一直选择，直到有序 )

2.2 算法实现

function selectSort(arr) {
    const len = arr.length;
    var minIndex = 0;

    for (let i = 0; i < len; i++) {
        minIndex = i;
        for (let j = i + 1; j < len; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }

        let temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp;
    }
    return arr;
}

2.3 算法分析

  • 最佳情况：T(n) = O(n^2)
  • 最差情况：T(n) = O(n^2)
  • 平均情况：T(n) = O(n^2)

3. 插入排序（Insert Sort）

3.1 算法概述

插入排序的代码实现虽然没有冒泡排序和选择排序那么简单粗暴，但它的原理应该是最容易理解的了，因为只要打过扑克牌或者麻将的人都应该能够秒懂。

插入排序在实现上，通常采用in-place排序（即只需用到O(1)的额外空间的排序），因而在从后向前扫描过程中，需要反复把已排序元素逐步向后挪位，为最新元素提供插入空间。

  • 从第一个元素开始，该元素可以认为已经被排序；
  • 取出下一个元素，在已经排序的元素序列中从后向前扫描；
  • 如果该元素（已排序）大于新元素，将该元素移到下一位置；
  • 重复步骤3，直到找到已排序的元素小于或者等于新元素的位置；
  • 将新元素插入到该位置后；
  • 重复步骤2~5。

3.2 算法实现

function insertSort(arr) {
    const len = arr.length;

    for (let i = 1; i < len; i++) {
        let temp = arr[i];
        for (var j = i - 1; j >= 0 && arr[j] > temp; j--) {
            //腾出位置给temp
            arr[j + 1] = arr[j];
        }
        //将temp插入
        arr[j + 1] = temp;
    }
    return arr;
}

3.3 算法分析

  • 最佳情况：T(n) = O(n)
  • 最坏情况：T(n) = O(n2)
  • 平均情况：T(n) = O(n2)

3.4 算法优化

  • 折半插入排序

    由于前半部分为已排好序的数列，这样我们不用按顺序依次寻找插入点，可以采用折半查找的方法来加快寻找插入点的速度。

    即不要每次都从已排序队列的尾部进行向前查找，而是使用折半搜索，寻找插入位置

    时间复杂度不变，但比较次数减少

4. 希尔排序（Shell Sort）

4.1 算法概述

希尔排序是插入排序的一种更高效率的实现，也称为缩小增量排序。
它与插入排序的不同之处在于，它会优先比较距离较远的元素。
希尔排序的核心在于间隔序列的设定，优先比较距离较远的元素。
既可以提前设定好间隔序列，也可以动态的定义间隔序列。

  • 先将要数组按某个增量d（一般增量取n/2，n为要排序数的个数）分成若干组
  • 所有距离为d的倍数的记录放在同一个组中，在各组内进行直接插入排序；
  • 然后再用一个较小的增量（d/2）对它进行分组，每组再进行直接插入排序；
  • 直至增量减为1，进行直接插入排序后，整体排序完成

4.2 算法实现

function shellSort(arr) {
    const len = arr.length;
    var d = Math.floor(arr.length / 2);

    while (d > 1) {
        //插入排序  
        for (let i = d; i < len; i++) {
            let temp = arr[i];
            for (var j = i - d; j >= 0 && arr[j] > temp; j = j - d) {
                arr[j + d] = arr[j];
            }
            arr[j + d] = temp;
        }
        //缩小增量
        d = Math.floor(d / 2);
    }
    return arr;
}

4.3 算法分析

  • 最佳情况：T(n) = O(nlog2 n)
  • 最坏情况：T(n) = O(nlog2 n)
  • 平均情况：T(n) = O(nlog2 n)

5. 堆排序（Heap Sort）

5.1 算法概述

堆排序（Heapsort）是指利用堆这种数据结构所设计的一种排序算法。堆积是一个近似完全二叉树的结构，并同时满足堆积的性质。

大顶堆：每个节点的值都大于或等于其子节点的值，在堆排序算法中用于升序排列；
小顶堆：每个节点的值都小于或等于其子节点的值，在堆排序算法中用于降序排列。

  • 首先将长度为n的序列构建称为大顶堆，此时根节点一定是当前序列的最大值；
  • 取出当前大顶堆的根节点，将其与序列末尾元素进行交换；
  • 对交换后的n-1个序列元素进行调整，使其满足大顶堆的性质；
  • 重复b、c两个步骤，直至堆中只有1个元素为止。

5.2 算法实现

function heapSort(arr) {
    var len = arr.length;
    // 建立大顶堆
    for (let i = Math.floor(len / 2); i >= 0; i--) {
        heapify(arr, i);
    }

    for (let i = len - 1; i > 0; i--) {
        //  输出堆顶元素，并移动到序列末尾
        [arr[0], arr[i]] = [arr[i], arr[0]];
        len--;
        // 重新调整堆
        heapify(arr, 0);
    }
    return arr;

    // 调整大顶堆（函数的闭包）
    function heapify(arr, i) {
        var left = 2 * i + 1;
        var right = 2 * i + 2;
        var largest = i;

        if (left < len && arr[left] > arr[largest]) {
            largest = left;
        }
        if (right < len && arr[right] > arr[largest]) {
            largest = right;
        }
        //  如果堆发生了调整，应再次递归调整其子二叉树
        if (largest !== i) {
            [arr[i], arr[largest]] = [arr[largest], arr[i]];
            heapify(arr, largest);
        }
    }
}

5.3 算法分析

  • 最佳情况：T(n) = O(nlogn)
  • 最差情况：T(n) = O(nlogn)
  • 平均情况：T(n) = O(nlogn)

6. 快速排序（Quick Sort）

6.1 算法概述

快速排序是对冒泡排序的一种改进，也可以说是在冒泡排序基础上的递归分治法。

  • 从数列中挑出一个元素，称为 “基准”（pivot）；
  • 重新排序数列，所有元素比基准值小的摆放在基准前面，所有元素比基准值大的摆在基准的后面（相同的数可以到任一边）。在这个分区退出之后，该基准就处于数列的中间位置。这个称为分区（partition）操作；
  • 递归地（recursive）把小于基准值元素的子数列和大于基准值元素的子数列排序。

6.2 算法实现

function quickSort(arr, low, high) {
    if (low < high) {
        var index = partition(arr, low, high);

        quickSort(arr, low, index - 1);
        quickSort(arr, index + 1, high);
    }
    return arr;

    function partition(arr, low, high) {
        //  取分组的第一个元素为基准
        const pivot = arr[low];
        while (low < high) {
            //  high下标的元素比pivot基准小，挪到pivot的左侧
            while (low < high && arr[high] >= pivot) {
                high--;
            }
            arr[low] = arr[high];

            //  low下标的元素比pivot基准大，挪到pivot的右侧
            while (low < high && arr[low] <= pivot) {
                low++;
            }
            arr[high] = arr[low];
        }
        //  low==high跳出循环，把pivot基准放到中间
        arr[low] = pivot;
        return low;
    }
}

6.3 算法分析

快速排序的最坏运行情况是O(n²)，比如说顺序数列的快排。
但它的平摊期望时间是O(nlogn) ，且O(nlogn)记号中隐含的常数因子很小，比复杂度稳定等于O(nlogn)的归并排序要小很多。所以，对绝大多数顺序性较弱的随机数列而言，快速排序总是优于归并排序

快速排序的内循环比大多数排序算法都要短小，这意味着它无论是在理论上还是在实际中都要更快。
它的主要缺点是非常脆弱，在实现时要非常小心才能避免低劣的性能。

6.4 算法优化

  • 基准的选择

    1. 固定基准

      默认取序列的第一个为基准

    2. 随机基准

      取待排序列中任意一个元素作为基准

      function randomPivot(arr, low, high) {
 //生成 [low,high] 的随机整数
 const pivot = parseInt(Math.random() * (high - low + 1) + low, 10);
 [arr[pivot], arr[low]] = [arr[low], arr[pivot]];
}

    3. 三数取中

      虽然随机选取枢轴时，减少出现不好分割的几率，但是还是最坏情况下还是O(n^2），要缓解这种情况，就引入了三数取中选取枢轴.

      使用左端low、右端high 和中心位置mid 上的三个元素的中位数作为基准

      function middlePivot(arr, low, high) {
 const mid = (high - low) / 2 + low;
 //使用三数取中法选择枢轴
 if (arr[mid] > arr[high]) {
     //目标: arr[mid] <= arr[high]
     [arr[mid], arr[high]]=[arr[high], arr[mid]]
 }
 if (arr[low] > arr[high]) {
     //目标: arr[low] <= arr[high]
     [arr[low], arr[high]]=[arr[high], arr[low]]
 }
 if (arr[mid] > arr[low]) {
     //目标: arr[low] >= arr[mid]
     [arr[mid], arr[low]]=[arr[low], arr[mid]]
 }
}

  • 当待排序序列的长度分割到一定大小后，使用插入排序

    原因：对于很小和部分有序的数组，快排不如插排好。当待排序序列的长度分割到一定大小后，继续分割的效率比插入排序要差，此时可以使用插排而不是快排

    // 分区小到一定程度时，用其他排序会更快
if (high - low + 1 < 2) {
insertSort(arr, low, high);
return;
}

7. 归并排序（Merge Sort）

7.1 算法概述

归并排序也是一种典型的分而治之思想的算法应用；

和选择排序一样，归并排序的性能不受输入数据的影响，但表现比选择排序好的多，因为始终都是O(nlogn）的时间复杂度，代价是需要额外的内存空间；

归并排序是一种稳定的排序方法。

  • 把长度为n的输入序列分成两个长度为n/2的子序列；
  • 对这两个子序列分别递归采用归并排序，直至子序列长度为1；
  • 将两个排序好的子序列合并成一个最终的排序序列。

7.2 算法实现

function mergeSort(arr) {
    const len = arr.length;
    // 直到长度为1时，停止递归
    if (len < 2) {
        return arr;
    }
    var mid = Math.floor(len / 2);
    var left = arr.slice(0, mid);
    var right = arr.slice(mid);

    return merge(mergeSort(left), mergeSort(right));

    function merge(left, right) {
        var res = [];
        while (left.length > 0 && right.length > 0) {
            // shift()： 删除arr第一个元素，并将其返回
            if (left[0] <= right[0]) {
                res.push(left.shift());
            } else {
                res.push(right.shift());
            }
        }
        // 把left与right剩余的元素，一起并入res的尾部
        return res.concat(left, right);
    }
}

7.3 算法分析

  • 最佳情况：T(n) = O(n)
  • 最差情况：T(n) = O(nlogn)
  • 平均情况：T(n) = O(nlogn)
  • 空间复杂度：O(n)

8. 基数排序（Radix Sort）

8.1 算法概述

基数排序是非比较的排序算法

基数排序有两种方法：

  1. MSD 从高位开始进行排序
  2. LSD 从低位开始进行排序
  • 将所有待比较元素（正整数）统一为同样的数位长度，数位较短的数前面补零；
  • 从个位开始，进行排序；
  • 然后进一位，再进行排序；直至最高位完成排序。

8.2 算法实现

function radixSort(arr, maxDigit) {
    // maxDigit 表示最大数字的位数
    var counter = [];
    // 以十进制进行排序
    var mod = 10;
    var dev = 1;
    // 初始化[0,9]的篮子
    for (let i = 0; i < 10; i++) {
        counter[i] = [];
    }

    for (let i = 0; i < maxDigit; i++, dev *= 10, mod *= 10) {
        for (let j = 0; j < arr.length; j++) {
            var bucket = parseInt((arr[j] % mod) / dev);
            counter[bucket].push(arr[j]);
        }
        var index = 0;
        for (let j = 0; j < counter.length; j++) {
            const len = counter[j].length;
            for (let k = 0; k < len; k++) {
                // shift()会改变目标数组的长度，所以要另存len
                arr[index++] = counter[j].shift();
            }
        }
    }
    return arr;
}

8.3 算法分析

基数排序 vs 计数排序 vs 桶排序：

  • 基数排序：根据键值的每位数字来分配桶
  • 计数排序：每个桶只存储单一键值
  • 桶 排 序：每个桶存储一定范围的数值
  • 最佳情况：T(n) = O(n * k)
  • 最差情况：T(n) = O(n * k)
  • 平均情况：T(n) = O(n * k)
  • k：“桶”的个数
